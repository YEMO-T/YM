## 🎯 需求详解与完成验证

### 需求 1️⃣ : 大模型根据与用户的多轮对话与知识库生成 PPT

#### ✅ 完成情况
- **状态：** 全部完成（包括 Bug 修复）
- **涉及文件：** `backend/service/llm_service.py`
- **关键改动：** 1 处（初始化 `context_str`）

#### 技术实现
```
多轮对话 → RAG 检索 → 构造上下文 → LLM 生成 PPT
```

**流程说明：**
1. 用户输入课题及历史对话
2. 后端自动调用 `get_knowledge_items(user_id)` 获取用户知识库
3. 通过 `_build_retrieval_query()` 从多轮对话中提取检索词
4. `_retrieve_top_k_knowledge()` 检索最相关的 Top 5 资料
5. `_format_knowledge_context()` 将资料格式化为上下文
6. 将上下文注入系统提示词发送给 LLM
7. LLM 基于上下文生成详尽的 PPT（8-10 页，>1500 字）

#### 验证方法
```bash
# 1. 上传包含课题相关内容的知识库资料
# 2. 执行: curl -X POST 'http://localhost:8000/api/coursewares/generate' \
#   -H 'Content-Type: application/json' \
#   -d '{"prompt": "生成PPT", "history": [], "user_id": "test_user"}'
# 3. 检查返回的 JSON 中 slides 是否包含知识库资料的内容
```

### 需求 2️⃣ : 工作台自动记录对话，至多 30 轮

#### ✅ 完成情况
- **状态：** 已完成（功能已存在，验证无误）
- **涉及文件：** `backend/repository/supabase_client.py`、`src/components/chat/DashboardView.tsx`
- **关键改动：** 0 处（代码已存在，无需修改）

#### 技术实现
```
用户发送消息 → insert_message() 保存 → trim_messages() 自动裁剪 → 保留最新 60 条
```

**流程说明：**
1. 每次用户或 AI 回复，调用 `insert_message(user_id, role, content)`
2. 插入成功后自动调用 `trim_messages(user_id, limit=60)`
3. 查询该用户所有消息 ID，按时间倒序排列
4. 删除超过 60 条的旧消息
5. 前端加载时调用 `getChatHistory(user_id)` 获取最新 60 条消息

#### 验证方法
```bash

# 2. 检查数据库: SELECT COUNT(*) FROM messages WHERE user_id='xxx'
# 期望结果: 60（或更少，如果有其他用户）
# 3. 刷新工作台，应该显示最新的 30 轮对话
```

#### ✅ 完成情况

- **涉及文件：** `backend/api/knowledge.py`、`src/App.tsx`
- **关键改动：** 0 处（代码已存在，无需修改）

#### 技术实现

**后端权限隔离：**
```python
# 所有操作都使用 .eq("user_id", user_id) 进行权限检验
GET /api/knowledge?user_id=xxx    # 只返回该用户的资料
POST /api/knowledge/upload         # 自动关联到 user_id
DELETE /api/knowledge/{id}         # 只能删除自己的资料
PUT /api/knowledge/{id}            # 只能编辑自己的资料
```

**前端用户界面：**
| 功能 | 实现位置 | 说明 |
|------|---------|------|
| 显示列表 | 表格（~560 行） | 显示用户所有资料 |
| 编辑功能 | 编辑模态框 | 修改名称、标签、内容 |
| 删除功能 | 删除确认 | 显示确认对话 |
| 上传功能 | 文件选择器 | 自动转为 .txt 格式 |

#### 数据隔离验证
```
用户 A: 上传资料 A1、A2 → 只能看到 A1、A2
用户 B: 上传资料 B1、B2 → 只能看到 B1、B2 + 无法看到 A 的资料
用户 A: 不能删除 B1、B2 → 权限检验通过数据库 WHERE 子句实现
```

#### 验证方法
```bash
# 1. 账户 A：上传知识库资料并生成 PPT
# 2. 账户 B：登录，应该看不到账户 A 的资料和对话
# 3. 账户 A：编辑资料，修改应该立即生效
# 4. 账户 A：删除资料，应该出现确认对话
# 5. 检查数据库：每个用户的对话和资料完全隔离
```
